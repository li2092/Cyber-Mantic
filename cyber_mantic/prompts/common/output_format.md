# 输出格式规范

## 通用规范

1. **编码**：UTF-8
2. **换行**：使用 `\n` 作为换行符
3. **Markdown**：支持基本Markdown格式

## 文本长度控制

| 场景 | 建议长度 |
|------|----------|
| 初判描述 | 50-80字 |
| 综合分析 | 100-150字 |
| 多理论汇总 | 200-300字 |
| 最终报告 | 400-800字 |
| 单个建议 | 20-50字 |

## 结构化输出

当需要结构化输出时，使用JSON格式：

```json
{
  "type": "response|analysis|report",
  "content": "主要内容",
  "metadata": {
    "confidence": 0.8,
    "source_theories": ["八字", "梅花"]
  }
}
```

## 追问格式

追问应该：
1. 使用有序列表
2. 每个问题独立成行
3. 可选项标注"（可选）"

示例：
```
请您提供以下信息：
1. 出生日期（年月日）
2. 性别
3. MBTI类型（可选，如不知道可跳过）
```

## 回溯问题格式

回溯问题应该：
1. 清晰指向过去事件
2. 简单易答（是/否/简短描述）
3. 不涉及过于私密的信息

示例：
```
为了验证分析准确性，请回答几个问题：

1. 在过去3年内，您是否经历过工作上的重大变动？
2. 2023年左右，是否有一次重要的决策让您印象深刻？
3. 您是否感觉最近1年人际关系有明显变化？

可以简单回答是/否，或补充一些细节。
```

## 报告格式

最终报告使用Markdown格式：

```markdown
## 标题

### 一级章节

**重点内容**加粗显示

- 列表项1
- 列表项2

### 行动建议

1. 建议1
2. 建议2
3. 建议3

---
*结尾署名或免责声明*
```

## 吉凶表达

| 等级 | 表达方式 |
|------|----------|
| 大吉 | "大吉之象"、"甚为有利" |
| 小吉 | "总体向好"、"有利因素较多" |
| 平 | "中平之象"、"需要观望" |
| 小凶 | "需要谨慎"、"有阻碍但可克服" |
| 大凶 | "需要特别注意"、"暂时不宜行动" |

注意：避免使用过于绝对或令人恐惧的表达。
